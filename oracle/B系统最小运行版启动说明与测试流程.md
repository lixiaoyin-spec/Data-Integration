# B 系统最小运行版启动说明 + 测试流程

本文档用于说明 Oracle 版 B 系统的最小运行版本如何启动、如何配置、如何验证功能是否正常。

---

## 一、系统组成

B 系统最小运行版由以下三部分组成：

1. **Oracle 数据库脚本**
   - 文件：`oracle_b_schema.sql`
   - 作用：创建表、约束、视图、触发器、存储过程，并插入测试数据。

2. **本地服务器**
   - 文件：`BLocalServer.java`
   - 作用：连接 Oracle，提供登录、查课、选课、退课、成绩单等接口，并内置前端页面。

3. **浏览器前端页面**
   - 作用：通过本地服务器提供的页面进行交互测试。

---

## 二、运行前准备

### 1. 环境要求

- JDK 8 或更高版本
- Oracle 数据库已启动
- Oracle JDBC 驱动可用
- 本地网络端口未被占用（默认使用代码中配置的端口）

### 2. 需要确认的数据库信息

在 `BLocalServer.java` 中检查并修改以下配置：

- Oracle JDBC 地址
- 数据库用户名
- 数据库密码
- 端口号
- SID 或服务名

如果你的 Oracle 环境不同，需要把连接字符串改成对应格式。

---

## 三、数据库初始化步骤

### 步骤 1：执行建表脚本

在 Oracle 客户端中执行：

- `oracle_b_schema.sql`

执行后应完成以下内容：

- 创建 `B_STUDENT`
- 创建 `B_ACCOUNT`
- 创建 `B_COURSE`
- 创建 `B_ENROLLMENT`
- 创建视图
- 创建存储过程
- 创建触发器
- 插入测试数据

### 步骤 2：确认数据是否写入成功

可执行以下查询检查：

```sql
SELECT * FROM B_STUDENT;
SELECT * FROM B_COURSE;
SELECT * FROM B_ENROLLMENT;
SELECT * FROM B_ACCOUNT;
```

如果能查到数据，说明初始化成功。

---

## 四、启动本地服务器

### 方式说明

`BLocalServer.java` 是本地服务主程序。启动后应提供：

- 登录接口
- 课程接口
- 选课接口
- 退课接口
- 成绩单接口
- 内嵌前端页面

### 启动步骤

1. 确认 Oracle 数据库已启动
2. 确认 `oracle_b_schema.sql` 已执行完成
3. 启动 `BLocalServer.java`
4. 查看控制台是否出现启动成功信息
5. 打开浏览器访问本地服务地址

---

## 五、前端页面访问方式

启动成功后，在浏览器打开本地服务器地址。

页面通常包含以下区域：

- 登录区域
- 操作区域
- 课程列表区域
- 成绩单区域

如果页面可以正常显示，说明前端已正常加载。

## 六、测试流程

下面给出建议的最小测试流程。建议按顺序测试。

---

### 测试 1：页面是否正常打开

#### 操作
- 启动本地服务器
- 在浏览器访问页面

#### 预期结果
- 页面正常显示
- 能看到登录框、课程按钮、选课按钮等

#### 判定
- 页面能打开：通过
- 页面无法打开：检查服务是否启动、端口是否冲突

---

### 测试 2：登录验证

#### 测试账号

可使用初始化脚本中的账号：

- 账户名：`b2023001`
- 密码：`123456`

#### 操作
- 输入账号和密码
- 点击“登录验证”

#### 预期结果
- 返回登录成功
- 页面状态栏显示当前用户信息

#### 判定
- 成功：说明 Oracle 查询与登录接口正常
- 失败：检查账户表数据、用户名密码、数据库连接

---

### 测试 3：查询全部课程

#### 操作
- 点击“全部课程”

#### 预期结果
- 显示课程列表
- 每门课程应包含课程号、名称、学时、学分、老师、地点、共享标记等信息

#### 判定
- 能正确展示：通过
- 无数据显示：检查课程表数据和接口返回 XML

---

### 测试 4：查询共享课程

#### 操作
- 点击“共享课程”

#### 预期结果
- 只显示 `SHARE_FLAG = 'Y'` 的课程

#### 判定
- 过滤正确：通过
- 数据不对：检查共享标记字段

---

### 测试 5：查询学生列表

#### 操作
- 点击“学生列表”

#### 预期结果
- 显示所有学生信息
- 包含学号、姓名、性别、专业、年级、电话、状态

#### 判定
- 能正常展示：通过

---

### 测试 6：学生选课

#### 操作
1. 输入学号，例如 `B2023003`
2. 输入课程号，例如 `C001`
3. 点击“选课”

#### 预期结果
- 选课成功
- `B_ENROLLMENT` 中新增一条记录
- 若课程容量满，会返回错误提示

#### 判定
- 成功插入：通过
- 若重复选课：应提示“该学生已选该课程”
- 若容量满：应提示“课程容量已满，无法选课”

---

### 测试 7：退课

#### 操作
1. 输入已有选课记录对应的学号和课程号
2. 点击“退课”

#### 预期结果
- 对应选课记录状态变成 `DROPPED`
- 页面提示退课成功

#### 判定
- 成功更新：通过
- 未找到记录：应返回错误提示

---

### 测试 8：查询成绩单

#### 操作
1. 输入学号，例如 `B2023001`
2. 点击“查成绩单”

#### 预期结果
- 显示该学生的课程、成绩、选课状态、选课时间等信息

#### 判定
- 数据完整展示：通过
- 无记录时应显示空结果或提示信息

---

## 七、数据库侧验证点

如果前端显示正常，但你想进一步确认后端是否正确，可以在 Oracle 中执行：

### 1. 验证选课结果

```sql
SELECT * FROM B_ENROLLMENT WHERE SID = 'B2023003';
```

### 2. 验证共享课程

```sql
SELECT * FROM V_B_SHARED_COURSE;
```

### 3. 验证成绩单视图

```sql
SELECT * FROM V_B_STUDENT_SCORE WHERE SID = 'B2023001';
```

### 4. 验证课程选课统计

```sql
SELECT * FROM V_B_COURSE_ENROLL_COUNT;
```

## 八、常见问题排查

### 1. 页面打不开

可能原因：
- 本地服务器未启动
- 端口被占用
- 代码中访问地址不正确

### 2. 登录失败

可能原因：
- Oracle 连接失败
- 账户表没有对应数据
- 用户名或密码不一致

### 3. 查不到课程

可能原因：
- 课程表没有初始化数据
- 查询接口 SQL 有误
- XML 返回格式解析失败

### 4. 选课失败

可能原因：
- 已重复选课
- 课程容量已满
- 学生或课程不存在
- 存储过程未正确创建

### 5. 数据库脚本执行报错

可能原因：
- 表已存在
- 先后顺序不正确
- 触发器或存储过程依赖未满足

解决方法：
- 清理已有对象后重新执行
- 按脚本顺序执行

## 九、最小运行版的验收标准

只要满足下面这些条件，就可以认为 B 系统最小运行版可用：

1. Oracle 脚本能成功执行
2. 本地服务器能启动
3. 浏览器页面能打开
4. 能正常登录
5. 能查询课程
6. 能执行选课
7. 能执行退课
8. 能查看成绩单
