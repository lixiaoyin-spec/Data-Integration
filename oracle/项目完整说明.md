# Oracle 侧（B 学院教务系统）完整说明

本目录是实验三中 B 学院（Oracle）教务系统的完整实现。B 学院采用 Oracle 数据库，通过本地 HTTP 服务对外提供 JSON/XML 接口，并支持与集成服务器的跨库联动。

---

## 系统架构

B 系统由三层组成：

1. **Oracle 数据库** — 核心表、视图、触发器、存储过程，管理学生/课程/选课数据
2. **本地 HTTP 服务** (`BLocalServer.java`) — 连接 Oracle，提供 REST 风格接口，内嵌前端页面
3. **浏览器前端** — 内嵌于 Java 服务中的单页 HTML，支持登录、查询、选课、退课、成绩单

## 数据库设计

### 核心表

| 表名 | 说明 | 关键字段 |
|---|---|---|
| `B_STUDENT` | 学生表 | SID, SNAME, SEX, MAJOR, PASSWORD, GRADE, PHONE, STATUS |
| `B_ACCOUNT` | 账户表 | ACC_NAME, ACC_PWD, LEVEL_NO, SID（外键关联 B_STUDENT） |
| `B_COURSE` | 课程表 | CID, CNAME, HOURS, CREDIT, TEACHER, LOCATION, SHARE_FLAG, CAPACITY, STATUS |
| `B_ENROLLMENT` | 选课表 | SID + CID 联合主键, SCORE, ENROLL_DT, CHOICE_STA |

### 视图

- `V_B_SHARED_COURSE` — 共享课程视图（SHARE_FLAG = 'Y'）
- `V_B_STUDENT_SCORE` — 学生成绩单视图（关联三表）
- `V_B_COURSE_ENROLL_COUNT` — 课程选课人数统计视图

### 触发器

- `TRG_B_ENROLL_CAPACITY` — 选课容量限制（超出 CAPACITY 时拒绝插入）
- `TRG_B_SCORE_STATUS` — 成绩录入时自动维护选课状态

### 存储过程

| 过程名 | 说明 |
|---|---|
| `P_B_ENROLL_COURSE(sid, cid)` | 学生选课（含重复检查） |
| `P_B_DROP_COURSE(sid, cid)` | 学生退课（状态改为 DROPPED，不物理删除） |
| `P_B_UPDATE_SCORE(sid, cid, score)` | 录入/更新成绩 |
| `P_B_GET_TRANSCRIPT(sid, cursor)` | 查询学生成绩单 |
| `P_B_GET_SHARED_COURSE(cursor)` | 查询共享课程列表 |
| `P_B_GET_COURSE_ENROLL(cid, cursor)` | 查询某课程选课情况 |
| `P_B_EXPORT_TRANSCRIPT_XML(sid, cursor)` | 导出成绩单 XML |
| `P_B_IMPORT_ENROLL_XML(sid, cid)` | 外部选课 XML 导入落库 |
| `P_B_GET_OPEN_COURSE(cursor)` | 查询开放选课的课程 |

### 初始化数据规模

- 学生：50 人（B2023001 ~ B2023050）
- 课程：10 门（C001 ~ C010），其中奇数编号课程为共享课程
- 选课：每人 5 门，总计约 250 条记录
- 账户：管理员 `badmin` + 50 个学生账户

## XML 与协议层

### XSD 定义

- `xml/shared_courses.xsd` — 共享课程结构：`<SharedCourses college="B">` 含多个 `<Course>` 元素，每个 Course 包含 cid, cname, hours, credit, teacher, location, shareFlag, capacity, status
- `xml/enrollment_exchange.xsd` — 选课交换结构：`<enrollments>` 含多个 `<enrollment>` 元素（sid + cid）

### XSLT 转换（双向）

| 方向 | 文件 |
|---|---|
| B 课程 → 统一格式 | `xslt/courseBToUniform.xsl` |
| B 选课 → 统一格式 | `xslt/enrollmentBToUniform.xsl` |
| B 学生 → 统一格式 | `xslt/studentBToUniform.xsl` |
| 统一格式 → B 课程 | `xslt/uniformToCourseB.xsl` |
| 统一格式 → B 选课 | `xslt/uniformToEnrollmentB.xsl` |
| 统一格式 → B 学生 | `xslt/uniformToStudentB.xsl` |

### 集成契约

`integration/b_exchange_contract.xml` 定义了 B 学院的集成端点、Payload Schema 引用和交换规则：
- 共享课程通过 XML 导出
- 跨库选课通过 XML 提交 enrollment 节点
- 退课保留记录并变更状态，不物理删除

## 快速启动

### 1. 初始化数据库

在 Oracle 客户端（SQL*Plus、SQL Developer 等）中以 BUSER 身份执行：

```sql
@db/oracle_b_schema.sql
```

脚本会自动完成建表、创建视图/触发器/存储过程、插入 50 学生 / 10 课程 / 每人 5 门课的初始化数据。

### 2. 确认数据落库

```sql
SELECT COUNT(*) FROM B_STUDENT;     -- 应 >= 50
SELECT COUNT(*) FROM B_COURSE;      -- 应 >= 10
SELECT COUNT(*) FROM B_ENROLLMENT;  -- 应 >= 250
SELECT COUNT(*) FROM B_ACCOUNT;     -- 应 >= 51
```

### 3. 检查并修改数据库连接配置

编辑 `service/BLocalServer.java` 第 38-41 行，确认以下参数与实际 Oracle 环境一致：

```java
OracleBRepository repository = new OracleBRepository(
    "jdbc:oracle:thin:@localhost:1521/orcl",  // JDBC 连接串
    "BUSER",                                    // 用户名
    "BPASS"                                     // 密码
);
```

### 4. 编译并启动服务

```bash
javac -cp ".;ojdbc8.jar" service/BLocalServer.java
java -cp ".;ojdbc8.jar" BLocalServer
```

控制台输出 `B Local Server started on port 8082` 即启动成功。

### 5. 访问前端页面

浏览器打开 `http://localhost:8082`，可看到 B 系统教务管理页面。

## 验收标准

- 学生数量 >= 50，课程数量 >= 10
- 每个学生至少 5 条选课记录
- 共享课程数量 > 0
- 选课记录无孤立引用（无坏的外键）
- 退课功能可将状态改为 `DROPPED`
- 本地服务能正常启动，前端页面可访问
- 登录、选课、退课、成绩单功能正常
- XML 导出/导入接口正常响应

## 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|---|---|---|
| 页面打不开 | 服务未启动 / 端口被占用 | 检查控制台输出，确认 8082 端口 |
| 登录失败 | Oracle 连接失败 / 账户数据不存在 | 检查数据库连接串、用户名、密码；确认脚本已执行 |
| 查不到课程 | 课程表无数据 / SQL 有误 | 执行 `SELECT * FROM B_COURSE` 确认 |
| 选课失败 | 重复选课 / 容量已满 / 存储过程未创建 | 检查报错信息，确认触发器和存储过程已创建 |
| 脚本执行报错 | 表已存在 / 执行顺序不对 | 先 DROP 已有对象，再按脚本顺序执行 |
