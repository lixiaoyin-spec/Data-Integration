## 三、跨院选课模块

### 3.1 请求与响应

**请求体**（JSON 或 form-urlencoded）:

```json
{
  "sno": "S001",
  "snm": "张三",
  "sex": "男",
  "sde": "计算机科学",
  "source_college": "A",
  "cno": "C001",
  "cnm": "数据库原理",
  "target_college": "B"
}
```

**成功响应（跨院）:**

```json
{
  "status": "success",
  "message": "跨院选课成功: 学生S001已选修学院B的课程数据库原理",
  "source_college": "A",
  "target_college": "B",
  "steps": [
    {"step": "check_source_college", "result": {"online": true, "...": "..."}},
    {"step": "check_target_college", "result": {"online": true, "...": "..."}},
    {"step": "import_student", "format": "xml", "result": {"status": "success"}},
    {"step": "import_enrollment", "format": "xml", "result": {"status": "success"}}
  ]
}
```

**失败响应示例:**

| 场景 | HTTP | message |
|------|------|---------|
| 缺少参数 | 400 | 缺少必要参数: sno, snm, source_college, cno, target_college |
| 无效学院 | 400 | 无效的学院 |
| 源学院离线 | 503 | 源学院A服务器不可用 |
| 目标学院离线 | 503 | 目标学院B服务器不可用 |
| 学生导入失败 | 400 | 学生导入失败: … |
| 选课写回失败 | 400 | 选课失败: … |
| 服务器异常 | 500 | 选课失败: … |

### 3.2 处理流程（跨院 vs 本院）

```
POST /api/enroll
{ sno, snm, sex, sde, source_college, cno, cnm, target_college }
         │
         ▼
┌─────────────────────────────────────┐
│ Step 1: check_source_college 在线    │
│         check_target_college 在线    │
│         任一离线 → 503 + steps       │
├─────────────────────────────────────┤
│ source_college != target_college ?   │
└──────────────┬──────────────────────┘
               │
     ┌─────────┴─────────┐
     │ 跨院               │ 本院
     ▼                    ▼
┌──────────────────┐  ┌──────────────────┐
│ Step 2:          │  │ Step 2:          │
│ build_student_xml│  │ (跳过导入)        │
│ import_student_xml│  │                  │
│ → POST /x/students/import (XML)       │
├──────────────────┤  ├──────────────────┤
│ Step 3:          │  │ Step 2:          │
│ build_enrollment_xml                  │
│ import_enrollment_xml                 │
│ → POST /x/enrollments/import (XML)    │
│                  │  │ client.enroll()  │
│                  │  │ → POST /x/enroll  │
└──────────────────┘  └──────────────────┘
         │                    │
         └────────┬───────────┘
                  ▼
         返回 { status, message, steps }
```

**关键技术点：**
- **跨院选课**（`source != target`）：学生信息与选课记录均以 **标准 XML** 写回课程所属学院（`target_college`）
- **本院选课**（`source == target`）：不经过 import，直接 `POST /x/enroll`（form: `sid`/`cid`）
- 每一步结果写入 `steps`，任一步失败即中止并返回具体错误
- `source_college` / `target_college` 自动转大写

### 3.3 与退选模块的关系

| 对比项 | 跨院选课 `/api/enroll` | 跨院退选 `/api/drop`（成员4） |
|--------|------------------------|------------------------------|
| 复杂度 | 多步（导入学生 + 写回选课） | 单步转发 |
| 数据格式 | 跨院：**XML** 写回 import 端点 | JSON/form 转发 drop |
| 目标 | 目标学院**创建**学生与选课 | 目标学院**删除**选课 |
| 学院客户端 | `import_student_xml()` + `import_enrollment_xml()` | `drop()` |

## 四、XML 写回模块

### 4.1 统一集成格式（本模块构建）

与成员2定义的标准格式一致，由 `xml_utils.py` 在 Python 侧直接构建（等价于 XSLT 统一层输出）：

**学生 XML**（`build_student_xml`）:

```xml
<?xml version="1.0" encoding="utf-8"?>
<Students>
  <student>
    <id>S001</id>
    <name>张三</name>
    <sex>男</sex>
    <major>计算机科学</major>
  </student>
</Students>
```

| 输入字段 | XML 标签 |
|----------|----------|
| `sno` | `<id>` |
| `snm` | `<name>` |
| `sex` | `<sex>` |
| `sde` | `<major>` |

**选课 XML**（`build_enrollment_xml`）:

```xml
<?xml version="1.0" encoding="utf-8"?>
<Choices>
  <choice>
    <sid>S001</sid>
    <cid>C001</cid>
  </choice>
</Choices>
```

| 输入字段 | XML 标签 |
|----------|----------|
| `sno` | `<sid>` |
| `cno` | `<cid>` |

### 4.2 写回传输

```
IntegrationServer                    目标学院 (A/B/C)
─────────────────                    ─────────────────
build_student_xml(data)
        │
        ▼
import_student_xml(xml)  ──POST──►  /x/students/import
        Content-Type: application/xml; charset=utf-8

build_enrollment_xml(data)
        │
        ▼
import_enrollment_xml(xml) ──POST──► /x/enrollments/import
        Content-Type: application/xml; charset=utf-8
```

`college_client._post_xml()` 负责设置 XML 请求头、UTF-8 编码及 JSON 响应解析。

### 4.3 与成员2 XSLT 的关系

| 方式 | 实现 | 本模块选用 |
|------|------|------------|
| XSLT 转换 | `enrollment_to_import.xsl` + `xsltproc` | 可选扩展 |
| Python 构建 | `build_student_xml` / `build_enrollment_xml` | **已采用** |

本模块在集成层用 `ElementTree` 直接生成统一格式 XML，学院端 import 接口同时兼容 XML/JSON；若需按学院原生 schema 二次转换，可叠加成员2的 `uniformToStudentB.xsl` 等脚本。

## 五、学院端 import 接口

### 5.1 A 学院（SQL Server — `local_server.py`）

| 端点 | 支持格式 | 行为 |
|------|----------|------|
| `POST /a/students/import` | XML / JSON / form | 解析 `id`/`sno`、`name`/`snm`、`sex`、`major`/`sde`；已存在则跳过 |
| `POST /a/enrollments/import` | XML / JSON / form | 解析 `sid`/`sno`、`cid`/`cno`；已存在则跳过 |

XML 解析：`_parse_student_import_xml()`、`_parse_enrollment_import_xml()`（`ElementTree`）

### 5.2 B 学院（Oracle — `BLocalServer.java`）

| 端点 | 支持格式 | 行为 |
|------|----------|------|
| `POST /b/students/import` | XML / JSON / form | `importStudent()`：查 `B_STUDENT` 后插入 |
| `POST /b/enrollments/import` | XML / JSON / form | `importEnrollmentSkipExisting()`：查重后 `enrollCourse()` |

XML 解析：`parseStudentXml()`、`parseEnrollmentXml()`（正则提取标签，与 A 标签名兼容）

### 5.3 C 学院（MySQL — 待成员6）

`config.py` 已配置 `/c/students/import`、`/c/enrollments/import`。C 学院 API 服务器需参照 A 的 `local_server.py` 实现同等 XML 接收能力后，方可参与 XML 写回联调。

## 六、代码结构

### 6.1 集成层（main.py）

```python
from xml_utils import build_student_xml, build_enrollment_xml

# 跨院分支
student_xml = build_student_xml({...})
target_client.import_student_xml(student_xml)

enroll_xml = build_enrollment_xml({"sno": sno, "cno": cno})
target_client.import_enrollment_xml(enroll_xml)
```

### 6.2 学院客户端（college_client.py）

```
CollegeClient
  ├── _post_xml(key, xml_str)           # XML POST 通用方法
  ├── import_student_xml(xml_str)       # → POST /x/students/import
  ├── import_enrollment_xml(xml_str)    # → POST /x/enrollments/import
  ├── import_student(dict | 位置参数)    # JSON 路径（保留，供其他调用）
  ├── import_enrollment(sno, cno)       # JSON 路径（保留）
  ├── enroll_course(enroll_data)        # 本院选课 form 路径（保留）
  └── enroll(sid, cid)                  # 本院选课（main 本院分支使用）

CollegeRegistry
  └── get(college_id)                   # 获取目标学院客户端
```

### 6.3 XML 工具（xml_utils.py）

```
build_student_xml(student_data)     # dict → Students/student XML 字符串
build_enrollment_xml(enroll_data)   # dict → Choices/choice XML 字符串
  └── 均通过 _pretty_xml() 输出 UTF-8 声明 + 缩进
```

### 6.4 学院端（A / B）

```
CollegeA — local_server.py
  ├── _parse_student_import_xml()
  ├── _parse_enrollment_import_xml()
  ├── _handle_import_student()      # 增加 XML 分支
  └── _handle_import_enrollment() # 增加 XML 分支 + 选课去重

BLocalServer — BLocalServer.java
  ├── parseStudentXml() / parseEnrollmentXml()
  ├── handleImportStudent()
  ├── handleImportEnrollment()
  └── OracleBRepository.importEnrollmentSkipExisting()
```

## 十一、技术决策说明

| 决策 | 选择 | 原因 |
|------|------|------|
| 跨院写回协议 | XML（`application/xml`） | 满足作业 XML 技术要求；与三学院数据交换格式统一 |
| XML 构建方式 | Python `ElementTree` + `_pretty_xml()` | 无外部 xsltproc 依赖；与 `parse_*_xml()` 对称 |
| 本院选课 | 仍用 `POST /x/enroll` | 学生已在本地库，无需 import |
| 学院端兼容 | XML + JSON + form 并存 | 联调灵活；集成层固定走 XML |
| 重复数据 | 学院端查重后跳过 | 避免重复插入选课/学生；返回 `status: success` |
| 失败策略 | 分步 `steps` + 遇错即停 | 便于验收定位失败环节 |
| 客户端分层 | `build_*`（集成层）+ `import_*_xml`（传输层） | 职责清晰，便于单测与复用 |
| B 学院实现 | Java 正则解析 XML | 与现有 `importEnrollmentsFromXml` 风格一致 |
| A 学院实现 | Python `ElementTree` 解析 | 与项目其他 XML 处理一致 |
| JSON 接口保留 | `import_student()` / `import_enrollment()` 未删除 | 不影响其他调用方；跨院主流程专用 XML |
