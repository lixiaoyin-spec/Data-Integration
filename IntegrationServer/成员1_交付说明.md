## 二、集成服务器 API 总览（13 个端点）

端口：**8000**

### 系统管理
| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/` | Web 仪表盘（内嵌HTML前端） |
| GET | `/api/status` | 检查集成服务器及所有学院在线状态 |
| GET | `/api/colleges` | 返回所有学院的注册信息和运行状态 |

### 认证
| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/login` | 统一登录：接收 `{acc, pwd, college}` → 转发到对应学院验证 |

### 数据聚合
| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/courses/shared` | 从所有在线学院拉取**共享课程**（share_flag=Y），统一XML格式返回。支持 `?college=B` 筛选 |
| GET | `/api/courses/all` | 从所有在线学院拉取**全部课程** |
| GET | `/api/courses/college?college=A` | 获取特定学院的课程 |
| GET | `/api/students` | 从所有在线学院拉取全体学生，统一XML格式返回 |
| GET | `/api/students/college?college=A` | 获取特定学院的学生 |

### 跨院选课（作业核心）
| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/enroll` | **跨院选课**：① 验证源/目标学院在线 → ② 将学生信息导入目标学院 → ③ 在目标学院创建选课记录 |
| POST | `/api/drop` | **跨院退选**：同步删除目标学院的选课记录 |

### 统计与查询
| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/statistics` | 全局统计：汇总三学院学生总数、课程总数、选课总数、各学院明细 |
| GET | `/api/transcript?sno=S001&college=A` | 获取学生在指定学院的成绩单（XML格式） |

## 三、跨院选课核心流程

```
POST /api/enroll
{
  "sno": "S001",
  "snm": "张三",
  "sex": "男",
  "sde": "计算机科学",
  "source_college": "A",    ← 学生所属学院
  "cno": "C001",
  "cnm": "数据库原理",
  "target_college": "B"     ← 课程所属学院
}
         │
         ▼
┌─────────────────────────────────────┐
│ Step 1: 检查 source_college 和      │
│         target_college 是否在线      │
├─────────────────────────────────────┤
│ Step 2: POST /b/students/import     │
│         → 将学生信息写入学院B数据库   │
│         （如已存在则跳过）            │
├─────────────────────────────────────┤
│ Step 3: POST /b/enroll              │
│         → 在学院B创建选课记录         │
├─────────────────────────────────────┤
│ 返回: { status: "success",          │
│         message: "跨院选课成功",     │
│         steps: [...] }              │
└─────────────────────────────────────┘
```

关键技术点：
- **本院选课**（source == target）直接转发，不经过导入流程
- **跨院选课** 先将学生信息写入目标学院，再创建选课记录
- 每个步骤独立执行并记录结果，失败时返回具体出错步骤

## 五、启动方式

```powershell
# 1. 安装依赖
pip install -r IntegrationServer/requirements.txt

# 2. 启动 A 学院本地 API 服务器（端口 8081）
python CollegeA_SQLServer/local_server.py --port 8081

# 3. 启动 B 学院本地服务器（端口 8082）
# 运行 BLocalServer.java

# 4. 启动 C 学院本地服务器（端口 8083）
# 待成员6创建后启动

# 5. 启动集成服务器（端口 8000）
python IntegrationServer/main.py --port 8000
```

浏览器打开 `http://localhost:8000` 即可访问集成仪表盘。

## 八、技术决策说明

| 决策 | 选择 | 原因 |
|------|------|------|
| 服务器框架 | Python `http.server`（内置库） | 零外部依赖，无需安装 Flask |
| HTTP 客户端 | `requests` 库 | 比 `urllib` 更简洁健壮，仅一个额外依赖 |
| 数据格式 | XML（标准集成格式） | 作业要求采用 XML 技术，A 系统已定义好 schema |
| 前端 | 内嵌 HTML + 原生 JS | 开箱即用，无需构建工具，供成员5参考或替换 |
| 格式统一 | 每个 `parse_*_xml()` 函数 | B 的 XML 标签名与 A 有差异（如 `score` vs `credit`），统一层处理差异 |
