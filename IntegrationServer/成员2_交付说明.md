## 二、课程聚合模块（CourseAggregator）

### 2.1 类结构

```
CourseAggregator(registry)
  ├── fetch_all(colleges)      # 从指定学院拉取全部课程
  ├── fetch_shared(colleges)   # 从指定学院拉取仅共享课程
  ├── normalize()              # 字段名统一化（处理异构schema）
  ├── deduplicate()            # 课程名相似度去重（阈值75%）
  ├── filter_shared()          # 筛选仅共享课程
  ├── filter_college(cid)      # 筛选指定学院课程
  ├── statistics()             # 返回聚合统计
  ├── to_xml(courses)          # 导出统一格式XML
  └── to_dict_list(courses)    # 导出dict列表
```

### 2.2 异构字段统一映射

三学院数据库schema不同，课程字段各异。聚合模块自动识别并统一：

| 统一字段 | A(SQL Server) | B(Oracle) | C(MySQL) |
|----------|---------------|-----------|----------|
| cno | Cno / id | id / cid | Cno |
| cnm | Cnm / name | name / cname | Cnm |
| credit | Cpt / credit | score / credit | Cpt |
| hours | Ctm / hours | time / hours | Ctm |
| teacher | Tec / teacher | teacher | Tec |
| location | Pla / location | location | Pla |
| share_flag | Share / share | shareFlag / share | Share |

### 2.3 去重策略

- 基于课程名称的 `SequenceMatcher` 相似度计算（去除括号内容后比较）
- 相似度 ≥ 75% 视为同一门课
- 同名课程合并时保留所有开课学院信息（`cross_college: true`, `offered_by: [A,B,C]`）

### 2.4 新增API端点

**`GET /api/courses/aggregated`**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `shared_only` | `0` | 设为 `1` 仅返回共享课程 |
| `college` | 无 | 筛选指定学院（A/B/C） |

响应：
- `Accept: application/json` → JSON（含 statistics + courses 列表）
- 默认 → XML（统一格式，含去重合并信息）

## 三、XSLT转换脚本

### 3.1 完整覆盖矩阵

三学院均已完成 **原生 ↔ 统一格式** 的双向XSLT覆盖：

| 数据 | 方向 | A | B | C |
|------|------|----|----|-----|
| 课程 | 原生→统一 | `formatClass.xsl` | `courseBToUniform.xsl` ✨ | `courseCToUniform.xsl` |
| 课程 | 统一→原生 | `classToA.xsl` | `uniformToCourseB.xsl` ✨ | `uniformToCourseC.xsl` ✨ |
| 学生 | 原生→统一 | `formatStudent.xsl` | `studentBToUniform.xsl` ✨ | `studentCToUniform.xsl` |
| 学生 | 统一→原生 | `studentToA.xsl` | `uniformToStudentB.xsl` ✨ | `uniformToStudentC.xsl` ✨ |
| 选课 | 原生→统一 | `formatClassChoice.xsl` | `enrollmentBToUniform.xsl` ✨ | `choiceCToUniform.xsl` |

> ✨ = 成员2新增

### 3.2 College B (Oracle) XSLT 关键映射

B系统原始XML格式与统一格式的主要差异：

**课程映射：**
```
B原生:  <id> <name> <time> <score> <teacher> <location> <shareFlag> <capacity> <status>
   ↓ courseBToUniform.xsl ↓
统一:   <id> <name> <hours> <credit> <teacher> <location> <share_flag> <share_college>
   ↓ uniformToCourseB.xsl ↓
B原生:  <id> <name> <time> <score> <teacher> <location> <shareFlag> <capacity=60> <status=OPEN>
```

**学生映射：**
```
B原生:  <id=SID> <name=SNAME> <sex=SEX> <major=MAJOR> <grade> <phone> <status>
   ↓ studentBToUniform.xsl ↓
统一:   <id> <name> <sex> <major>
   ↓ uniformToStudentB.xsl ↓
B原生:  <id> <name> <sex> <major> <grade=2024> <phone/> <status=ACTIVE>
```

**选课映射：**
```
B原生:  <enrollments><enrollment><sid> <cid></enrollment></enrollments>
   ↓ enrollmentBToUniform.xsl ↓
统一:   <Choices><choice><sid> <cid> <score/></choice></Choices>
```

### 3.3 College C (MySQL) 反向XSLT

C系统原有 `studentCToUniform.xsl`、`courseCToUniform.xsl`、`choiceCToUniform.xsl`（原生→统一），缺少反向转换。本次补充：

**课程反向：**
```
统一:   <Classes><class><id> <name> <credit> <hours> <teacher> <location> <share_flag> <share_college>
   ↓ uniformToCourseC.xsl ↓
C原生:  <Courses><course><Cno> <Cnm> <Cpt> <Ctm> <Tec> <Pla> <Share>
```

**学生反向：**
```
统一:   <Students><student><id> <name> <sex> <major>
   ↓ uniformToStudentC.xsl ↓
C原生:  <Students><student><Sno> <Snm> <Sex> <Sde> <Pwd=id>
```

**选课反向：**
```
统一:   <Choices><choice><sid> <cid> <score>
   ↓ uniformToChoiceC.xsl ↓
C原生:  <Choices><choice><Sno> <Cno> <Grd>
```

### 3.4 集成服务器级XSLT

| 脚本 | 输入 | 输出 | 用途 |
|------|------|------|------|
| `filter_shared.xsl` | 全部课程XML | 仅共享课程XML | 课程共享过滤 |
| `enrollment_to_import.xsl` | 统一选课XML | 目标学院导入XML | 跨院选课时转换选课数据到目标学院格式（参数 `target_college=A/B/C`） |
| `courses_to_summary.xsl` | 全部课程XML | 课程汇总统计XML | 生成课程总数/共享数/共享课程清单 |

## 四、统一集成格式（XML Schema）

三学院XSLT转换的目标格式——集成服务器统一XML格式：

### 课程 (Classes)
```xml
<Classes>
  <class>
    <id>C001</id>           <!-- 课程号 -->
    <name>数据库原理</name>    <!-- 课程名 -->
    <credit>4</credit>       <!-- 学分 -->
    <hours>64</hours>        <!-- 学时 -->
    <teacher>张老师</teacher>  <!-- 教师 -->
    <location>A101</location> <!-- 上课地点 -->
    <share_flag>Y</share_flag>     <!-- 是否共享 Y/N -->
    <share_college>A</share_college> <!-- 来源学院 -->
  </class>
</Classes>
```

### 学生 (Students)
```xml
<Students>
  <student>
    <id>S001</id>            <!-- 学号 -->
    <name>张三</name>          <!-- 姓名 -->
    <sex>男</sex>              <!-- 性别 -->
    <major>计算机科学</major>    <!-- 专业 -->
  </student>
</Students>
```

### 选课 (Choices)
```xml
<Choices>
  <choice>
    <sid>S001</sid>          <!-- 学号 -->
    <cid>C001</cid>          <!-- 课程号 -->
    <score>85</score>         <!-- 成绩 -->
  </choice>
</Choices>
```

## 五、其他成员依赖本模块的方式

### 成员3（跨院选课含XML写回）
- 课程数据拉取后，可用 XSLT 转换为目标学院原生格式再写入
- 选课导入：调用 `enrollment_to_import.xsl` 并设置 `target_college` 参数
- 例如将统一选课XML转为B学院可导入格式：`xsltproc --stringparam target_college B enrollment_to_import.xsl enrollment.xml`

### 成员4（退选+统计）
- 统计接口可直接使用 `CourseAggregator.statistics()` 获得去重后的跨院课程统计
- 课程汇总可用 `courses_to_summary.xsl` 生成统计视图

### 成员1（集成服务器主程序）
- `CourseAggregator` 已集成到 `main.py`，新增 `/api/courses/aggregated` 端点
- 聚合模块可独立调用，也可通过 `aggregate_shared_courses(registry)` / `aggregate_all_courses(registry)` 便捷函数使用
- 所有XSLT脚本可被 Python `lxml` 或 Java `javax.xml.transform` 调用

### 成员6（C系统联调）
- C系统反向XSLT已就绪，C的HTTP API服务器在接收统一格式数据后，可直接用XSLT转为C原生格式写入MySQL

## 六、技术决策说明

| 决策 | 选择 | 原因 |
|------|------|------|
| 聚合模块语言 | Python | 与集成服务器同语言，可共用 CollegeRegistry 和 CollegeClient |
| 去重算法 | SequenceMatcher(≥75%) | 比精确匹配更鲁棒，能处理"数据库原理"vs"数据库原理(A)"等变体 |
| XSLT版本 | XSLT 1.0 | 兼容性最广，Java内置Transformer和Python lxml均支持 |
| XSLT编码 | UTF-8 | 统一编码避免跨平台乱码（C原有用GB2312，新脚本统一为UTF-8） |
| 统一格式字段 | credit/hours/share_flag/share_college | 与集成服务器 `xml_utils.py` 的 `build_unified_courses_xml()` 一致 |
| 去重放在Python而非XSLT | Python | XSLT 1.0的Muenchian分组仅能按精确值分组，无法做模糊匹配 |
