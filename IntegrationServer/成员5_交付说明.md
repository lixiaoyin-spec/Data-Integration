## 三、功能模块详解

### 3.1 登录窗口

- **功能**: 输入账号、密码，选择所属学院（A/B/C），调用集成服务器 `POST /api/login` 进行统一身份验证
- **API调用**: `POST /api/login` — body: `{acc, pwd, college}`
- **特点**: 不直接连接各学院服务器，统一通过集成服务器认证

### 3.2 系统状态（Tab 1）

- **全局统计卡片**: 实时显示在线学院数、学生总数、课程总数、选课总数
- **学院详情卡片**: 每个学院一张卡片，显示在线状态、DBMS类型、地址、学生/课程/选课数量
- **刷新按钮**: 手动刷新所有状态数据
- **API调用**: `GET /api/status` → `GET /api/colleges` → `GET /api/statistics`

### 3.3 课程管理（Tab 2）

- **三种课程模式**:
  - 共享课程: `GET /api/courses/shared`（可选 `?college=A/B/C` 筛选）
  - 全部课程: `GET /api/courses/all`（可选 `?college=A/B/C` 筛选）
  - 聚合去重: `GET /api/courses/aggregated`（可选 `?shared_only=1` 和 `?college=A/B/C` 筛选）
- **双视图切换**:
  - 表格视图: 解析 XML → Treeview 表格（课程号、课程名、学分、学时、教师、地点、来源学院、共享标志）
  - XML 视图: 格式化显示原始 XML 数据
- **学院筛选**: 下拉框选择特定学院或留空查看全部

### 3.4 学生管理（Tab 3）

- **全体学生**: `GET /api/students`（从所有在线学院拉取学生数据）
- **按学院筛选**: `GET /api/students/college?college=A`（查看特定学院学生）
- **双视图切换**: 表格视图（学号、姓名、性别、专业、学院）和 XML 视图

### 3.5 跨院选课（Tab 4）

- **表单字段**: 学号、姓名、性别、专业、课程号、课程名、所属学院、目标学院
- **必填项**: 学号、姓名、课程号、所属学院、目标学院（带 * 标记）
- **选课流程**:
  1. 调用 `POST /api/enroll`，body 为 JSON 格式
  2. 集成服务器自动处理: 验证学院在线 → 导入学生信息到目标学院 → 在目标学院创建选课记录
  3. 结果区域展示每一步执行状态（JSON 格式化显示）
- **同院选课**: 当 source_college == target_college 时弹出确认提示

### 3.6 跨院退选（Tab 5）

- **表单字段**: 学号、课程号、目标学院
- **退选流程**:
  1. 调用 `POST /api/drop`，body 为 JSON 格式
  2. 弹出确认对话框防止误操作
  3. 结果区域展示退选结果

### 3.7 统计查询（Tab 6）

- **全局汇总**: 在线学院、学生总数、课程总数、选课总数
- **学院明细表**: Treeview 表格（学院名、DBMS、状态、学生数、课程数、选课数）
- **双视图**: 格式化视图（表格+汇总文本）和 JSON 原始视图
- **API调用**: `GET /api/statistics`

### 3.8 成绩单查询（Tab 7）

- **查询参数**: 学号 + 学院
- **双视图**:
  - 表格视图: 解析成绩单 XML → Treeview（课程号、课程名、成绩、学分）
  - XML 视图: 格式化显示原始成绩单 XML
- **兼容性**: 同时兼容 A/B/C 三种成绩单 XML 格式（Transcript/enrollment/choice 标签）
- **API调用**: `GET /api/transcript?sno=xxx&college=A`

## 四、技术实现要点

### 4.1 HTTP 通信

使用 Python 内置 `urllib.request`（零外部依赖，与 `main.py` 使用标准库的设计理念一致）：

```python
def api_get(path, params=None):  # GET 请求，支持查询参数
def api_post(path, data=None, json_data=None):  # POST 请求，支持 form 和 JSON
```

- 超时设置: 10 秒
- 自动设置 `Accept: application/json` 请求头
- 统一错误处理: 返回 `(status_code, body)` 元组

### 4.2 XML 处理

使用 `xml.dom.minidom` 解析和格式化 XML：

- `fmt_xml(text)`: 将 XML 字符串格式化（缩进美化）显示
- `_render_course_tree(xml_str)`: 解析课程 XML → Tkinter Treeview
- `_render_student_tree(xml_str)`: 解析学生 XML → Tkinter Treeview
- 成绩单查询兼容三种标签格式（A/B/C 系统 XML 差异）

### 4.3 UI 设计

- 字体: Microsoft YaHei（微软雅黑），中文友好
- 配色: 蓝色主题（#2563eb），状态指示灯（绿色在线/红色离线）
- 布局: 操作栏 + 内容区的标准布局，每个标签页结构一致
- 窗口尺寸: 1100×720，自动居中

## 五、启动方式

```powershell
# 1. 启动集成服务器（端口 8000）
python IntegrationServer/main.py --port 8000

# 2. 启动集成端 GUI
python IntegrationServer/integration_gui.py
```

**注意**: 集成端 GUI 依赖集成服务器已启动。各学院 API 服务器（8081/8082/8083）也需启动才能获取数据。

## 七、技术决策说明

| 决策 | 选择 | 原因 |
|------|------|------|
| GUI 框架 | Python Tkinter + ttk | Python 内置标准库，零外部依赖，与 College A 的 gui_app.py 技术栈一致 |
| HTTP 客户端 | `urllib.request`（内置库） | 与 main.py 使用标准库的理念一致，无需安装 requests |
| XML 解析 | `xml.dom.minidom`（内置库） | 标准库即可满足 XML 解析和格式化需求 |
| 架构模式 | 独立 GUI 进程 + HTTP 通信 | 与集成服务器解耦，可独立启动/关闭，不影响服务器运行 |
| 视图模式 | 表格 + XML 双视图 | 表格便于快速浏览数据，XML 便于查看原始数据和调试 |
| 登录方式 | 通过集成服务器统一登录 | 与作业要求一致，各学院登录由集成服务器转发验证 |
| 跨院选课 | 表单 → JSON POST | 与 main.py 的 `/api/enroll` 端点数据格式一致 |
| 编码风格 | 与 College A gui_app.py 一致 | 保持项目内代码风格统一 |
