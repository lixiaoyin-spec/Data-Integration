# 数据集成作业三 — 现场演示指南

## 一、演示前准备（提前10分钟）

### 1. 确认 MySQL 在运行

```powershell
mysql -u root -p123456 -e "USE teaching_c; SELECT COUNT(*) FROM student; SELECT COUNT(*) FROM course; SELECT COUNT(*) FROM choice;"
```

```powershell

```

```powershell
cd "D:\Three_\数据集成\第三次作业\CollegeA_SQLServer"

```
预期输出：`学生: 50 条  课程: 10 条  选课: 250 条`

```powershell
pip show requests pymysql 2>&1 | Select-String "Name:|Version:"
```

- `CollegeA_SQLServer/init_database.py` — A的Students表（Sno, Snm, Sex, Sde, Pwd）
- `oracle/db/oracle_b_schema.sql` — B的B_STUDENT表（SID, SNAME, SEX, MAJOR, GRADE, PHONE）

> "我们用XML解决异构问题。每个学院把自己的数据转换成标准XML格式输出，集成服务器解析统一格式的XML进行聚合。同时还用XSD做格式校验，XSLT做格式转换。"

**第3层：集成服务器路由**

## 六、故障应急预案

### 端口被占用
```powershell
taskkill /f /im python.exe
```
然后重新启动。

### MySQL 连不上
```powershell
# 检查MySQL是否运行
Get-Service MySQL* 2>$null
# 或直接重连
mysql -u root -p123456 -e "SELECT 1"
```

### 集成服务器 stats 转圈
等待10秒会返回。或刷新浏览器后再试。如持续超时，可跳过此步直接演示操作功能。

### 某学院离线
只说"B 系统 Oracle 暂未启动，我们重点演示 A → C 的跨院选课"，直接操作即可。

## 七、演示完后

老师可能会问的问题，准备好答案：

**Q: 为什么要用 XML？**
> XML 是平台无关的数据交换格式。三个学院用三种不同数据库甚至不同编程语言（Python/Java），XML 作为中间格式屏蔽了底层差异。我们还用 XSD 做格式校验、XSLT 做格式转换。

**Q: 跨院选课怎么保证数据一致性？**
> 集成服务器采用"先导入学生、再创建选课"的两步流程。第一步失败则第二步不执行。目标学院的 API 使用 INSERT IGNORE 避免重复导入。

**Q: 如果某学院服务器宕机了怎么办？**
> 集成服务器只在请求时实时检查学院在线状态，离线学院不会阻塞整体服务，在线学院的数据正常聚合。

**Q: B系统为什么用Mock？**
> Oracle 需要 Docker 容器或本地安装，环境配置复杂。我们完整的 Oracle 建表脚本和 Java 代码已在仓库中，Mock服务器返回格式完全一致，用于集成链路的端到端验证。
